Sprint-0 Checklist

Complete these before opening the project in your IDE. Click to check off items.

  • Run /project:setup-claude-project — generate CTO agent
  • Create Claude Project — paste CTO system prompt + upload knowledge files
  • Write project README.md (replace this template)
  • Adapt folder structure to project type (SaaS / Library / CLI / API)
  • Fill docs/0k_PRD.md
  • Fill docs/01_ARCHITECTURE.md
  • Fill docs/03_MODULES.md initial registry
  • Customize AGENTS.md (Tier-1)
  • Fill MUST_READ_MODULE_REUSE.md import table
  • Resolve role prompt placeholders (role_cto.md, role_cpo.md)
  • Configure pyproject.toml / package.json
  • Fill CLAUDE.md placeholders
  • Review .claude/settings.local.json
  • Review .claude/GUIDE.md for project-specific updates
  • Draft Sprint-01 plan
  • Smoke-test all slash commands

Role System

Every agent operates under a canonical role tag. Roles define what you produce and what you never touch.

[CPTO]
Chief Product & Technology Officer. Produces sprint plans, design reviews, architecture docs, team briefings. Never produces source code.
/project:cpto
[DEV:module|BE]
Backend Developer. Produces implementation code, unit tests, module READMEs. Must follow module reuse protocol.
/project:dev-backend
[DEV:module|FE]
Frontend Developer. Produces components, pages, Playwright E2E tests. Must follow UI Kit tokens.
/project:dev-frontend
[QA]
QA Engineer. Owns test plans, regression gates, bug reports. Does NOT implement features.
/project:dev-qa
[CSO]
Chief Security Officer. Audits auth, encryption, PII handling, dependencies. Does NOT implement features.
/project:dev-cso
[DESIGNER]
ARIA (UX Creative Agent). Produces runnable UI code, SVG/CSS/Canvas. Never produces prose or wireframes.
/project:dev-ux
[DEV:devops]
DevOps Engineer. Manages deployment, CI/CD, infrastructure. Cannot modify business logic or tests.
/project:dev-devops
[FOUNDER]
Human operator (Avi). Final decision maker on all scope, priority, and tradeoffs. Approves irreversible changes.
human

Decision Rights

SituationAction
Reversible decisionMake it, document in decisions_log.md, move on
Irreversible decisionFLAG it, write a DR, propose options, wait for FOUNDER
Two options, both finePick the one that reuses more existing modules
Scope creep detectedFLAG immediately, propose what to cut
Quality concernRun a GBU review, score it, present findings
